# Backlog

Esta pasta recebe os backlogs detalhados por fase quando a `factory` ou a `phase-detailing` quebram o roadmap em entregas executaveis.

Sugestao de nomes:

- `fase-01.md`
- `fase-02.md`
- `fase-03.md`

Cada backlog deve apontar para:

- objetivo da fase;
- historias ou tarefas;
- criterios de aceite;
- dependencias;
- docs que precisam ser atualizados.
