# Perguntas de Entrevista — Mobile App

Use em rodadas de no máximo 4 perguntas cada.

---

## Rodada 1 — O usuário mobile

1. Em qual situação o usuário vai abrir o app? (no trabalho, em movimento, em casa)
2. O usuário vai usar o app com as duas mãos ou uma mão só? (impacta o design)
3. O app precisa funcionar sem internet? Quais funcionalidades?
4. O usuário vai usar o app todos os dias ou ocasionalmente?

## Rodada 2 — Funcionalidades core

1. O que o usuário faz nos primeiros 30 segundos após abrir o app?
2. Qual é a ação mais importante que o app permite fazer?
3. O app vai ter notificações? De que tipo?
4. O app vai usar câmera, GPS, biometria ou outros recursos do celular?

## Rodada 3 — Plataforma e distribuição

1. iOS, Android ou os dois?
2. O app vai ser publicado na App Store / Play Store ou distribuído internamente?
3. Vai ter versão web também ou é só mobile?
4. Qual é o celular mais básico que o usuário-alvo provavelmente usa?

## Rodada 4 — Conta e dados

1. O usuário precisa criar conta ou pode usar sem cadastro?
2. Os dados ficam só no celular ou sincronizam com servidor?
3. O que acontece se o usuário troca de celular?
4. Tem dados sensíveis que precisam de proteção extra? (biometria, localização, saúde)

# Perguntas de Entrevista — E-commerce

---

## Rodada 1 — O negócio

1. O que será vendido? Produto físico, digital ou serviço?
2. Quantos produtos/SKUs no catálogo inicial?
3. Quem é o comprador típico? Pessoa física ou empresa?
4. Já existe operação de vendas hoje? Por qual canal?

## Rodada 2 — Venda e pagamento

1. Quais formas de pagamento precisa aceitar? (PIX, boleto, cartão, parcelamento)
2. Vai ter frete? Calculado como? (tabela fixa, Correios, transportadora)
3. Vai ter cupom de desconto ou promoção?
4. O cliente pode comprar sem criar conta?

## Rodada 3 — Estoque e entrega

1. O estoque é controlado pelo sistema ou em outro lugar?
2. Quem faz a separação e envio dos pedidos?
3. O cliente recebe atualização de status do pedido?
4. Como funciona a troca ou devolução?

## Rodada 4 — Gestão

1. Quem vai gerenciar o catálogo e os pedidos?
2. Precisa de relatório de vendas? Quais métricas são importantes?
3. Vai integrar com sistema fiscal para emitir nota?
4. Tem marketplace (Mercado Livre, Amazon) que precisa sincronizar?

# Perguntas de Entrevista — Website

---

## Rodada 1 — Objetivo

1. Qual é o objetivo principal do site? (gerar leads, vender, informar, portfólio)
2. Quem é o visitante ideal? O que ele precisa encontrar no site?
3. O que o visitante deve fazer depois de visitar? (ligar, preencher formulário, comprar)
4. Existe site atual? O que está errado com ele?

## Rodada 2 — Conteúdo

1. Quais páginas o site precisa ter?
2. Quem vai criar e atualizar o conteúdo depois que o site estiver no ar?
3. Vai ter blog ou área de conteúdo frequente?
4. Tem fotos, vídeos ou materiais prontos ou precisa produzir?

## Rodada 3 — Funcionalidades

1. Precisa de formulário de contato ou orçamento?
2. Vai ter área restrita para clientes ou parceiros?
3. Precisa de integração com WhatsApp, chat ou CRM?
4. Vai ter e-commerce ou apenas vitrine sem venda online?

# Perguntas de Entrevista — Automação / Script

---

## Rodada 1 — O processo

1. Descreva passo a passo o que a automação vai fazer.
2. Com que frequência vai rodar? (manual, agendado, disparado por evento)
3. De onde vêm os dados de entrada?
4. Onde vai o resultado? (arquivo, email, sistema, banco de dados)

## Rodada 2 — Erros e exceções

1. O que deve acontecer se a automação falhar?
2. Alguém precisa ser avisado quando rodar (com sucesso ou com erro)?
3. Precisa de log do que foi processado?
4. Tem caso de borda que precisa de tratamento especial?

## Rodada 3 — Integrações

1. Quais sistemas externos a automação vai acessar?
2. Existe autenticação ou API key necessária?
3. Tem limite de volume ou rate limit nos sistemas acessados?
4. Precisa rodar em horário específico por alguma restrição?
